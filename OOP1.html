Object Oriented Programming Introduction

Concept Theory

OOP is programming that is based on real world, where we have objects. For example, I could create a dog object that has several different attributes and behaviours, which is modeled by OOP.

Class - A template for the creation of objects.

Object - A specific object of this class. For example, I can have a people class, but only one instance of you. I would then create an *insert your name here* object of the human class.

Attributes - Similar to the real world, objects can have attributes that describe them. For example, a student has a name, a certain mark and an average. We can model these attributes through the use of instance variables.

Behaviours - Objects can also have behaviours, for example my dog has the ability to bark or eat, which can be modeled through the use of methods. In the case below, the student has the ability to have their marks set.

Constructors - This is what creates the default values for the student, that can later be changed using the methods in the program.

Creating objects - In the main program, I then create the objects. In this example, I have created an array of student objects, and created 5 students named Jack, Liam, Avery, Evan and Sonia.

Constructors with parameters - When creatings objects, I can also assign the attribute values right then and there. In order for this to happen, there needs to be a second cunstructor that accepts parameters and assigns the parameters to the variables/attributes.

Objects with parameters - In the main method of the program, when creating a new object, I've added the string "Loki" and the weight 3.6. The program reads that, and inputs those values into the constructor.
